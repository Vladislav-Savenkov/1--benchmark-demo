Причины завершения задач мобильным пользователям

Справочник предназначен для хранения списка причин, по которым может быть завершена поставленная мобильному сотруднику задача. Задача может быть завершена как пользователем 1С, так и сотрудником из мобильного приложения. В конфигурации принято три вида завершения задачи:

  • Доступна при любом завершении в 1С - завершение задачи в связи появившимися новыми обстоятельствами. Например, в офис позвонил клиент и отказался от заказа. Может выполнять только пользователь 1С.
  • Доступна при нормальном завершении мобильным сотрудником - завершение задачи после ее непосредственного выполнения. Такое завершение может быть выполнено только после того, как сотрудник приступил к непосредственному выполнению поставленной задачи. Может быть выполнено только мобильным сотрудником в мобильном приложении.
  • Доступна при вынужденном завершении мобильным сотрудником - завершение задачи в связи появившимися обстоятельствами, мешающими приступить к выполнению поставленной задачи. Например, мобильный сотрудник попал в пробку или аварию. 
  • Доступна при отклонении мобильным сотрудником - отклонение задачи мобильным сотрудником. Фактически это отказ от выполнения задачи по конкретной причине. Отклонение может выполнять только сотрудник при помощи приложения

Для того, чтобы разделить причины по месту применения и по виду завершения при вводе причины в справочник необходимо указать возможное место применения (доступна в мобильном приложении, доступна в 1С) и вид завершения (доступна при вынужденном завершении, доступна при нормальном завершении, доступна при отклонении. Такой подход позволяет в дальнейшем применять фильтры при выборе причины.

Например, сотрудник получил на мобильное приложение новую задачу, но не может ее выполнить - так как находится в пробке. Это означает, что ему необходимо отказаться от выполнения задачи с указанием причины. Для этого сотрудник нажимает в мобильном приложении кнопку "Отказаться" и указывает одну из возможных причин. При этом на выбор предоставляются только причины доступные в мобильном приложении при отклонении. По такому же принципу фильтр будет срабатывать и в других местах.
