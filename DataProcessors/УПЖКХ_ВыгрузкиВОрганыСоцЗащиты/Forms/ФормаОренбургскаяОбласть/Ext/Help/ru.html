Оренбургская область

Обработка предназначена для выгрузки данных в отдел социальной защиты населения Оренбургской области за указанный период.

Перед началом выгрузки данных в обработке необходимо заполнить следующие параметры:
  • «Месяц выгрузки» - месяц, за который необходимо предоставить данные в ОСЗН (далее - Месяц выгрузки);
  • «Организация» - организация, от имени которой будет предоставлены данные за Месяц выгрузки в ОСЗН (далее - Основная организация);  
  • «Файл загрузки» - указывается путь к dbf-файлу, который получен из ОСЗН (далее - Файл загрузки) со следующей структурой:

№

Наименование поля

Значение поля

1 KODORG Код района ЭСРН
2 GKU_ID Код группы ЖКУ в ЭСРН
3 MES Первое число месяца выгрузки
4 ADR Полный адрес
5 INDEX Индекс
6 MO Муниципальное образование (Город или Район)
7 NASP Населенный пункт
8 UL Улица
9 DOM Дом
10 KORP Корпус
11 KV Квартира
12 SNILS СНИЛС
13 FAM Фамилия получателя МПС
14 IM Имя получателя МПС
15 OT Отчество получателя МПС
16 LS Лицевой счет (при его наличии в Учреждении)


При запуске обработки необходимо выполнить следующую последовательность операций:

1. Необходимо произвести предварительную настройку. Для вызова диалогового окна настройки необходимо нажать на кнопку "2. Настройка". 
   На форме настройки на закладке "Настройки соответствия услуг" необходимо установить соответствие услуг в ОСЗН и услуг в Базе.

Для каждой услуги в ОСЗН необходимо определить способ свертки тарифа:

  • «Максимум» - по данной услуге будет браться максимально возможное значение тарифа;
  • «Минимум» - по данной услуге будет браться минимально возможное значение тарифа;
  • «Усреднять» - по данной услуге будет браться среднее значение тарифа;
  • «Суммировать» - по данной услуге будет браться суммарное значение тарифа.

Также для каждой услуги в ОСЗН необходимо указать единицу измерения.

Для каждой услуги в Базе необходимо определить способ определения начислений:

  • «Общедомовая» - по данной услуге производятся общедомовые начисления по коллективным приборам учета;  
  • «Индивидуальная» - по данной услуге производятся начисления на лицевой счет по индивидуальным приборам учета;
  • «Индивидуальная и общедомовая» - прочие варианты начисления по услуге.

Для всех услуг в ОСЗН в обязательном порядке должно быть установлено соответствие с одной или несколькими услугами в базе, в противном случае данные по услуге из ОСЗН выгружены не будут.

На закладке «Дополнительные настройки» необходимо установить следующие настройки:

  • «Льготополучателей идентифицировать по:» - способ идентификации в базе льготополучателей из файла: "по номеру лицевого счета" - идентификация по наименованию/идентификатору лицевого счета; "по номеру лицевого счета и ФИО жильца" - идентификация по ФИО жильца и наименованию/идентификатору лицевого счета; "по адресу регистрации" - идентификация по ФИО жильца и адресу помещения, в котором открыт лицевой счет;
  • «Вид отапливаемой площади помещения» - ссылка на элемент справоника «Виды площадей». С учетом выбранного вида площади будет определяться соответствующий размер площади выгружаемого помещения, на основе которого будет заполнено поле «S_OT» файла выгрузки.


При последующем использовании обработки шаг с настройкой можно пропускать.

2. Если все вышеперечисленные операции выполнены, то можно приступить к этапу загрузки сведений из Файла загрузки. Для этого необходимо нажать на кнопку "3. Загрузить сведения из файла загрузки".

В процессе выполнения операции загрузки производится формирование таблицы загрузки на основной форме, а также срабатывают механизмы идентификации в соответствии с настройкой «Льготополучателей идентифицировать по:».

Если по каким–либо причинам идентификация лицевого счета не прошла, то будет выведено соответствующее сообщение с указанием Фамилии, Имени, Отчества и номера лицевого счета в Учреждении, а поле "Лицевой счет" будет подчеркнуто красным пунктиром. Для данной строки можно указать лицевой счет в базе вручную, либо оставить эту строку не идентифицированной. 
 
3. Для получения сведений о начислениях и задолженностях за Месяц выгрузки по Основной организации для идентифицированных лицевых счетов из Файла загрузки, необходимо нажать на кнопку "4. Заполнить данными из информационной базы".

По завершению выполнения операции, на основной форме будет заполнена таблица выгрузки, сведения из которой далее можно выгрузить в файл выгрузки. 

4. Далее можно приступить к этапу выгрузки данных в Файл выгрузки. Для этого необходимо нажать на кнопку "5. Выгрузить в файл".  
    Выгружаемый файл будет сохранен в каталог, указанный в настройке «Файл загрузки».

Формат файла выгрузки данных с услугами ЖКУ имеет следующую структуру: 

№

Наименование поля

Значение поля

1 GKU_ID Код группы ЖКУ в ЭСРН
2 FAM Фамилия получателя МПС
3 IM Имя получателя МПС
4 OT Отчество получателя МПС
5 ADR Полный адрес
6 S_OB Общая площадь
7 S_G Жилая площадь
8 S_OT Отапливаемая площадь
9 ETAZ_H Этажность дома
10 ETAZ_KV Занимаемый этаж
11 VIDZF Тип жилищного фонда 1-частный, 2 – муниципальный 3- государственный
12 TIP_POM Тип жилого помещения 1 – многоквартирный 2 – индивидуальный 3 - общежитие (коммунальная квартира)
13 KOLCHLS Количество зарегистрированных граждан (по месту жительства и по месту пребывания)
14 KOL_PR Количество временно проживающих (без регистрации)
15 KOL_PR_C С какого числа временно проживают
16 KOL_PR_D По какое число временно проживают
17 KOL_OPR Общее количество проживающих (13+14-18)
18 K_VR_OT Количество временно отсутствующих
19 K_VR_OTC С какого числа временно отсутствуют
20 K_VR_OTD По какое число временно отсутствуют
21 USLUGA Наименование услуги
22 TARIF Тариф за 1 ед.измерения
23 NORM_B Норматив коммун.услуги Базовый на 1 ед.изм. (чел.) (для жилищной=0, для Быт.отх.=1)
24 NORM_P Норматив потребления коммунальной услуги Повышенный (для жилищной=0)
25 SCH Наличие индивидуального прибора учета на коммун.усл. 1 -да 0 -нет
26 V_USLUG Объем потребленных услуг за отчетный месяц, норматив коммун.усл. или площадь, на которую начислена жилищная услуга (Быт.отх.=кол-во чел.)
27 IZM Единица измерения
28 SUM_NACH Начислено за отчетный месяц
29 SUM_PER Сумма перерасчета (корректировки)
30 SUM_OT С какого числа произведено перерасчет
31 SUM_PO По какое число произведен перерасчет
32 S_K_OPL Начислено к оплате (28+29+34)
33 SUMM_OPL Фактический платеж в текущем месяце
34 SUM_UP Сальдо на начало месяца (Задолженность/-аванс)
35 SOGL Наличие соглашения о реструктуризации задолженности 1-да 0-нет
36 SOGL_V Выполнение соглашения о реструктуризации задолженности 1-да 0-нет
37 NUMLS Л/сч в организации (заполнение при его наличии, в случае отсутствия его в файле Учреждения или проверка достоверности Поставщиком)
38 INN ИНН поставщика, предоставляющего данные сведения
39 OGRN ОГРН поставщика, предоставляющего данные сведения
40 NAZVOGR Сокращенное наименование организации-поставщика
41 PRIM Примечание
42 PRIZNAK_1 ОГРН организации (строки 43)
43 PRIZNAK Организация, осуществляющая управление МКД или исполнитель ЖКУ (кроме энергоснабжения) для индивид.жилого дома
