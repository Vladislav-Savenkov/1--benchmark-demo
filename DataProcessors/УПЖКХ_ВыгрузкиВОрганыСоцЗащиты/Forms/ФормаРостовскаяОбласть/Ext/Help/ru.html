Ростовская область

Обработка предназначена для выгрузки данных в отдел социальной защиты населения (далее - ОСЗН) Ростовской области за указанный период.

Данная обработка разработана в соответствии со следующими нормативными документами: Федеральный закон от 27.07.2010 № 210-ФЗ «Об организации предоставления государственных и муниципальных услуг», ст. 159 и 160 Жилищного кодекса Российской Федерации, постановление Правительства Российской Федерации от 14.12.2005 № 761 «О предоставлении субсидий на оплату жилого помещения и коммунальных услуг», постановления Правительства Ростовской области от 09.12.2011 № 212 «О порядке расходования субвенций, поступающих в областной бюджет из федерального бюджета на финансовое обеспечение расходов по оплате жилищно-коммунальных услуг, оказываемых отдельным категориям гражданам, а также на выплату единовременного пособия беременной жене военнослужащего, проходящего военную службу по призыву, и ежемесячного пособия на ребенка военнослужащего, проходящего военную службу по призыву», от 15.12.2011 № 232 «О расходовании средств областного бюджета на предоставление гражданам в целях оказания социальной поддержки субсидий на оплату жилых помещений и коммунальных услуг, материальной и иной помощи для погребения, мер социальной поддержки в соответствии с отдельными нормативными правовыми актами социальной направленности, а также об условиях и порядке их назначения»

 

Перед началом использования обработки необходимо заполнить следующие поля основной формы:

1.   «Месяц выгрузки» - месяц (далее – Месяц выгрузки), за который необходимо предоставить данные в ОСЗН;

2.   «Организация» - ссылка на элемент справочника «Организации», по которой необходимо предоставить данные в ОСЗН за Месяц выгрузки;

3.   «Файл загрузки» - путь к файлу (далее - Файл ОСЗН), полученному из ОСЗН в котором необходимо дозаполнить сведения о начислениях из информационной базы (файл формата *.DBF);

4.   Установить соответствия между услугами в ОСЗН и справочника «Услуги» в информационной базе. Для этого необходимо перейти на форму настроек, которая откроется при нажатии на кнопку «2. Настройка». При этом:

4.1.             Таблица «Услуги в ОСЗН» состоит из следующих колонок:

4.1.1.                «Код услуги ОСЗН» - содержит код услуги в ОСЗН (не рекомендуем корректировать коды вручную, это может ошибочно отразиться на выгружаемых данных);

4.1.2.                «Услуга в ОСЗН» - содержит наименование услуги ОСЗН. Предназначено для ручного редактирования пользователями;

4.1.3.                «Способ определения начислений» - содержит способы определения сведений о произведенных начислениях по услуге в ОСЗН. Значения для вывода в данной колонке доступны из следующего списка:

4.1.3.1.  «Общедомовой» - для услуг из справочника «Услуги», которые установлены в соответствие текущей услуге в ОСЗН, получать сведения только о начислениях на общедомовые нужды (по умолчанию заполняется для полей с префиксом «SOUT»),

4.1.3.2.  «Индивидуальный» - для услуг из справочника «Услуги», которые установлены в соответствие текущей услуге в ОСЗН, получать сведения только об индивидуальных начислениях (по умолчанию заполняется для полей с префиксом «SIN»),

4.1.3.3.  «Совокупный» - для услуг из справочника «Услуги», которые установлены в соответствие текущей услуге в ОСЗН, получать полные сведения о начислениях (с учетом индивидуальной и общедомовой составляющей). (устанавливается только вручную).

4.2.             В таблице «Услуги в базе» для текущей услуги в ОСЗН устанавливаются соответствующие услуги из справочника «Услуги». При этом:

4.2.1.                Для одной услуги в ОСЗН можно назначить произвольное количество различных услуг из справочника «Услуги».

4.2.2.                Одна услуга из справочника «Услуги» может быть установлена только одной услуге в ОСЗН.

4.3.             Первоначальное заполнение таблицы «Услуги в ОСЗН» происходит на основании считанных из Файла ОСЗН сведений о кодах услуг в ОСЗН. Автоматически будут заполнены колонки «Код услуги в ОСЗН» и «Способ определения начислений». Пользователю необходимо будет заполнить сведения в колонке «Услуга в ОСЗН».

4.4.             Сохранение установленных настроек происходит при нажатии кнопки «Сохранить и закрыть». 

При работе с обработкой использовать следующий порядок действий:

 

1.   Для загрузки сведений из Файла ОСЗН в таблицу обработки необходимо нажать кнопку «1. Загрузить из файла».

2.   Для заполнения в таблице обработки сведений о начислениях и оплатах за Месяц выгрузки необходимо нажать кнопку «3. Заполнить данными из базы». По льготникам, которых не удалось идентифицировать в базе, будут выведены соответствующие информационные сообщения. При заполнении полей Файла ОСЗН учитывать следующий порядок:

2.1.     Сведения о начислениях по услугам из справочника «Услуги», которые соответствуют текущей услуге из ОСЗН со способом определения начислений «Общедомовой», заполняются в полях типа «SOUT+<Код услуги в ОСЗН>».

2.2.     Сведения о начислениях по услугам в базе, которые соответствуют текущей услуге из ОСЗН со способом определения начислений «Индивидуальный» или «Совокупный», заполняются в полях типа «SIN+<Код услуги в ОСЗН>».

 

3.   Для выгрузки данных из таблицы обработки в файл ОСЗН необходимо нажать кнопку «4. Выгрузить в файл». При этом данные будут записаны в Файл ОСЗН, а исходный вариант Файла ОСЗН будет записан в новый файл, наименование которого формируется следующим образом: [Наименование Файла загрузки] + "_Исходный".

При выгрузке данных в Файле ОСЗН предусмотрено заполнение только тех полей, которые в структуре формата файла «[Наименование Файла загрузки].DBF» выделены зеленым цветом (см. таблицу ниже).

 

Структура Файла ОСЗН представлена ниже:


Формат файла "[Наименование Файла ОСЗН].dbf":

Наименование поля

Значение поля

PROVIDER

Код поставщика ЖКУ. Значение из соответствующего поля Файла ОСЗН

PROVNAME

Наименование поставщика ЖКУ. Значение из соответствующего поля Файла ОСЗН

ACCOUNT

Номер лицевого счета в системе поставщика. Значение из соответствующего поля Файла ОСЗН

ADDRESID

Идентификационный номер адреса. Значение из соответствующего поля Файла ОСЗН

KRAYN

Район. Значение из соответствующего поля Файла ОСЗН

KNASP

Город/Населенный пункт. Значение из соответствующего поля Файла ОСЗН

KYLIC

Улица. Значение из соответствующего поля Файла ОСЗН

NDOM

Дом. Значение из соответствующего поля Файла ОСЗН

NSTR

Строение. Значение из соответствующего поля Файла ОСЗН

NKORP

Корпус. Значение из соответствующего поля Файла ОСЗН

NKW

Квартира. Значение из соответствующего поля Файла ОСЗН

NKOMN

Комната. Значение из соответствующего поля Файла ОСЗН

NPOM

Помещение. Значение из соответствующего поля Файла ОСЗН

NDOPTER

Дополнительная территория. Значение из соответствующего поля Файла ОСЗН

NPLNSTR

Планировочная структура. Значение из соответствующего поля Файла ОСЗН

DTFACT

Месяц начислений к оплате абонента, предшествующий дате формирования Файла. Значение из соответствующего поля Файла ОСЗН.

SUMFIN

Итоговая сумма начислений для текущего идентифицированного лицевого счета, которые содержатся в полях, в наименованиях которых содержится префикс «SIN»

SUMFOUT

Итоговая сумма начислений для текущего идентифицированного лицевого счета, которые содержатся в полях, в наименованиях которых содержится префикс «SOUT».

MONTHDBT**

Первое число месяца следующего за месяцем возникновения задолженности, в формате ДДММГГГГ.

ROUT[N]

Признак, который заполняется следующим образом:

1. «Истина» - если в Отчетном месяце на идентифицированный лицевой счет произведены начисления по услуге из базы, соответствующей виду услуги [N] в ОСЗН, с учетом объема потребления, распределенного на общедомовые нужды.

2. «Ложь» - в противном случае.
SOUT[N]  Итоговая сумма, начисленная в Отчетном месяце на идентифицированный лицевой счет по услуге из базы, соответствующей виду услуги [N] в ОСЗН, с учетом объема потребления, распределенного на общедомовые нужды, без учета повышающих коэффициентов.
RIN[N]

Признак, который заполняется следующим образом:

1. «Истина» - если в Отчетном месяце на идентифицированный лицевой счет произведены начисления по услуге из базы, соответствующей виду услуги [N] в ОСЗН.

2. «Ложь» - в противном случае.
SIN[N] Итоговая сумма, начисленная в Отчетном месяце на идентифицированный лицевой счет по услуге из базы, соответствующей виду услуги [N] в ОСЗН, с учетом индивидуального потребления, без учета повышающих коэффициентов.
TARIF[N] Тариф по услуге из базы, соответствующей виду услуги [N] в ОСЗН.
RINXX Признак наличия начислений по услуге, отнесенной к виду услуг «ХХ». Устанавливается истина, если начисления больше 0. Не заполняется.
SINXX Сумма начислений к оплате абоненту за расходы по услуге, отнесенной к виду услуг «ХХ» в месяце, указанном в DTFACT, без учета повышающих коэффициентов к нормативам потребления коммунальных услуг. Не заполняется.
TARIFXX Тариф по услуге, отнесенной к виду услуг «ХХ». Не заполняется.
NOTE Автоматически не заполняется, заполняется вручную посредством занесения данных в специальное поле табличной части.
