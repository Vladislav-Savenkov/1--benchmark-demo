Задача мобильному сотруднику

Документ «Задача мобильному сотруднику» предназначен для постановки и дальнейшего контроля поручений выездным сотрудникам.

В левой части шапки документа можно указать дату и номер абстрактной заявки, по которой необходимо выполнить задачу-поручение. Поля не являются обязательными для заполнения, их следует заполнять только в тех случаях, когда есть необходимость установить связь между задачей, поставленной мобильному сотруднику, и заявкой, которая явилась основанием для выполнения этой задачи.

Распределение задачи

В правой верхней части документа необходимо указать исполнителя, дату и время выполнения работ. Исполнитель может быть подобран при помощи «Формы подборы исполнителя». Форму удобно использовать, при большом количестве исполнителей или, когда важна оперативность и поставленная задача должна быть отправлена исполнителю, который является активным в данный момент времени.

Если задача по ошибке была направлена не тому сотруднику, ее необходимо аннулировать, и создать новую.

Статусы задачи

Под блоком подбора исполнителя находится строка-гиперссылка, информирующая о текущем состоянии задачи. Строка состояние может содержать, как только статус (см. статусы ниже), так и причину перехода в статус (указывается для статусов "Отклонил" и "Завершил"). При нажатии на гиперссылку открывается форма "Информация о ходе выполнения работ", в которой подробно описан ход выполнения работ, а также предусмотренна возможность просмотра событий по задаче на карте.

В один момент времени задача может находится в одном из предусмотренных статусов:

  • Черновик - подготовлен черновик задачи. При этом задача не попадает в очередь на отправку исполнителю. На этой стадии в задачу еще можно вносить изменения.
  • Готова к отправке - задача готова к отправке, и находится в очереди на отправку. С этого момента задачу нельзя корректировать.
  • Отправлена - задача успешно отправлена сотруднику.
  • Получил - задач доставлена на мобильное устройство сотрудника.
  • Прочитал - сотрудник прочитал задачу на мобильном устройстве.
  • Принял - сотрудник принял задачу в работу.
  • Отклонил - сотрудник отказался от выполнения поставленной задачи, с указанием одной из причин отказа.
  • Выехал - сотрудник выехал для выполнения задачи по указанному в задаче адресу. Для перехода в статус сотрудник должен на мобильном устройстве нажать кнопку "Выехал".
  • Приехал - сотрудник прибыл по адресу, для выполнения задачи.  Для перехода в статус сотрудник должен на мобильном устройстве нажать кнопку "Приехал".
  • Завершил - сотрудник завершил задачу. В системе предусмотрено два типа завершения задачи: вынужденное завершение и нормальное.
    • Под вынужденным завершением работ подразумевается закрытие задачи после ее принятия сотрудником в работу, но до момента начала выполнения работ (т.е. после принятия задачи в работу появились обстоятельства, которые мешают приступить к выполнению задачи, например сотрудник заболел, встал в пробку, попал в аварию и т.д.)
    • Под нормальным завершением задачи понимается статус, присваиваемый после выполнения поставленной задачи (даже в том случае если не удалось достичь поставленной цели).

Для удобства работы список задач раскрашивается в соответствии с текущим статусом. Статусы задач обновляются в автоматическом режиме, с периодичностью заданной в настройках обмена. Периодичность обновления списка задач задается в настройках формы списка. Для контроля изменения статусов удобно использовать систему уведомлений.

Контактные данные, описание задачи

Далее следует блок описания задачи, который включает: наименование контрагент, подразделение, адрес выполнения задачи, контактное лицо, телефоны контактного лица. Ниже следует описание выполненной работы. Если информации о выполненной работе отсутствует, пользователь видит фразу "...не введено...". Результаты выполнения могут быть введены как на мобильном устройстве, так и при завершении задачи в 1С (см. ниже).

Корректировать контактные данные и описание задачи можно только пока задача находится в статусе "Черновик". Если после отправки задачи возникает необходимость проинформировать сотрудника об изменениях - используйте комментарии по задаче (см. ниже)

Обмен комментариями по задаче

В нижней части формы задачи отображается таблица с комментариями по задаче. Использование механизма комментариев позволяет оперативно взаимодействовать с мобильным сотрудником. Комментарии могут быть созданы как пользователем 1С и отправлены в сторону мобильного клиента, так и на мобильном устройстве с дальнейшем получением в 1С. Периодичность обмена задается в настройках подключения к сервису.

Для отслеживания новых комментариев удобно использовать встроенную систему уведомлений. Для получения подробной информации о статусе комментария (прочитан, отправлен, доставлен, место прочтения на карте (для комментариев, отправленных из 1С)) необходимо установить курсор на интересующем комментарий и нажать кнопку "Информация по комментарию".

Завершение задачи пользователем 1С

В системе предусмотрено завершение задачи в 1С. Возможность предусмотрена для ситуаций, когда поставленную задачу необходимо аннулировать из-за вновь появившихся обстоятельств. Для этого необходимо воспользоваться кнопкой "Завершить" в верхней левой части формы задачи.

При этом пользователь должен указать причину завершения и описать выполненные работы (не обязательное поле). После того, как задача завершена мобильному пользователю отправляется уведомление, о том, что задача аннулирована.
